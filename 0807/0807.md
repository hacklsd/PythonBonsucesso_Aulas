Aula 08/07 – Projeto CRUD com Django

Objetivo

Criar um projeto Django simples com um aplicativo (app), páginas HTML e estrutura inicial para um sistema CRUD (Create, Read, Update e Delete).

---

1) Criar um ambiente virtual

O ambiente virtual cria um espaço isolado para instalar as bibliotecas do projeto, evitando conflitos com outros projetos Python.

No terminal, execute:

python -m venv nome-do-ambiente

Exemplo:

python -m venv myvenv

---

2) Ativar o ambiente virtual

Antes de instalar qualquer biblioteca, ative o ambiente virtual.

Se estiver usando PowerShell

.\nome-do-ambiente\Scripts\Activate.ps1

Se estiver usando o Prompt de Comando (CMD)

.\nome-do-ambiente\Scripts\activate.bat

Quando estiver ativado, o nome do ambiente aparecerá no início da linha do terminal.

Exemplo:

(myvenv) C:\Projetos\crud>

---

3) Instalar o Django

Com o ambiente virtual ativo, instale o Django:

pip install django

Esse comando baixa e instala o framework Django no ambiente virtual.

---

4) Criar o projeto Django

Execute:

django-admin startproject crud .

O que esse comando faz?

- cria um projeto chamado crud;
- cria o arquivo manage.py;
- cria a pasta crud, contendo os arquivos de configuração do projeto.

Importante

O espaço seguido do ponto (" .") no final significa:

«"Crie o projeto na pasta atual."»

Sem o ponto, o Django criaria uma nova pasta para o projeto.

---

5) Configurar o projeto (opcional)

Abra o arquivo:

crud/settings.py

Altere:

LANGUAGE_CODE = 'pt-br'

TIME_ZONE = 'America/Sao_Paulo'

Essas configurações deixam o projeto em português e ajustam o fuso horário para o Brasil.

---

6) Criar um aplicativo (App)

Agora vamos criar o primeiro aplicativo do projeto.

Execute:

python manage.py startapp app

O que é um App?

Pense assim:

- Projeto = uma empresa.
- App = um setor dessa empresa.

Exemplo:

Projeto: Instagram

Apps:

- Login
- Perfil
- Mensagens
- Fotos
- Reels
- Stories

Cada app possui sua própria responsabilidade dentro do projeto.

---

7) Registrar o App no projeto

Abra novamente:

crud/settings.py

Procure pela lista:

INSTALLED_APPS = [

Adicione o nome do app:

INSTALLED_APPS = [
    'django.contrib.admin',
    'django.contrib.auth',
    'django.contrib.contenttypes',
    'django.contrib.sessions',
    'django.contrib.messages',
    'django.contrib.staticfiles',
    'app',
]

O que isso faz?

Informa ao Django que esse aplicativo faz parte do projeto.

Sem essa linha, o Django ignora completamente o app.

---

8) Criar as migrações

Execute:

python manage.py makemigrations

O que esse comando faz?

Verifica se existem alterações nos modelos ("models.py") e cria arquivos de migração.

Esses arquivos informam ao Django quais mudanças deverão ser feitas no banco de dados.

Caso ainda não existam modelos, é normal que nenhuma migração seja criada.

---

9) Aplicar as migrações

Execute:

python manage.py migrate

O que esse comando faz?

Cria ou atualiza as tabelas do banco de dados de acordo com as migrações existentes.

Na primeira execução, o Django cria as tabelas necessárias para o funcionamento do sistema.

---

10) Criar as páginas (Views)

No arquivo:

app/views.py

importe o comando "render":

from django.shortcuts import render

importe também as bibliotecas platform e subprocess para a função do sistema:

import platform
import subprocess


Depois crie as funções:

def home(request):
    return render(request, 'home/home.html')


def cadastro(request):
    return render(request, 'cadastro/index.html')


def usuarios(request):
    return render(request, 'usuarios/index.html')


def sistema(request):
    sistema = platform.system()
    
    if sistema == "Linux":
        # 1. Nome do Servidor
        nome = subprocess.check_output("hostname", shell=True).decode().strip()
        
        # 2. Processador (Modelo e quantidade de núcleos)
        proc_cmd = "lscpu | grep 'Model name' | cut -d: -f2"
        processador = subprocess.check_output(proc_cmd, shell=True).decode().strip()
        
        # 3. Memória (Exibe o total, usado e disponível em formato legível)
        memoria = subprocess.check_output("free -h", shell=True).decode()
        
        # 4. Status de Rede (Interfaces ativas e IPs)
        rede = subprocess.check_output("ip -br addr show", shell=True).decode()

    elif sistema == "Windows":
        # 1. Nome do Servidor
        nome = subprocess.check_output("hostname", shell=True).decode().strip()
        
        # 2. Processador
        proc_cmd = "wmic cpu get name /value"
        processador = subprocess.check_output(proc_cmd, shell=True).decode().replace("Name=", "").strip()
        
        # 3. Memória (Total visível pelo sistema)
        mem_cmd = "wmic computersystem get TotalPhysicalMemory /value"
        mem_raw = subprocess.check_output(mem_cmd, shell=True).decode().replace("TotalPhysicalMemory=", "").strip()
        # Converte bytes para GB de forma simples
        if mem_raw.isdigit():
            memoria = f"Total de Memória Física: {round(int(mem_raw) / (1024**3), 2)} GB"
        else:
            memoria = "Não foi possível ler a memória."
        
        # 4. Status de Rede (Configuração de IP detalhada)
        rede = subprocess.check_output("ipconfig", shell=True).decode(errors='ignore')
        
    else:
        nome = "Sistema Desconhecido"
        processador = "N/A"
        memoria = "N/A"
        rede = "N/A"

    context = {
        'sistema': sistema,
        'nome_servidor': nome,
        'processador': processador,
        'memoria': memoria,
        'rede': rede
    }

    return render(request, 'sistema/index.html', context)



O que é uma View?

Uma View é uma função responsável por responder uma requisição do navegador.

Quando alguém acessa uma URL, a View decide qual página HTML será exibida.

---

11) Criar as rotas (URLs)

No arquivo:

crud/urls.py

importe as Views:

from app import views

Adicione em "urlpatterns":

path('', views.home, name='home'),
path('home/', views.home, name='home'),
path('cadastro/', views.cadastro, name='cadastro'),
path('usuarios/', views.usuarios, name='usuarios'),
path('sistema/', views.sistema, name='sistema'),

O que são rotas?

As rotas ligam um endereço da aplicação a uma função do arquivo "views.py".

Exemplo:

/home

↓

executa

views.home()

↓

mostra

home.html

---

12) Criar a pasta de Templates

Dentro da pasta app, crie a seguinte estrutura:

app/

└── templates/
    ├── home/
    ├── cadastro/
    ├── usuarios/
    └── sistema/

---

13) Criar os arquivos HTML

Crie os seguintes arquivos:

home/
    home.html

cadastro/
    index.html

usuarios/
    index.html

sistema/
    index.html

Esses arquivos serão exibidos pelas Views.

---

14) Criar o conteúdo das páginas

Em cada arquivo HTML, crie a estrutura básica da página e um menu de navegação com links entre elas.

Exemplo de menu:

- Home
- Cadastro
- Usuários
- Sistema

Assim será possível navegar entre todas as páginas da aplicação.

---

15) Executar o servidor

Execute:

python manage.py runserver

O Django iniciará um servidor local.

Abra o navegador e acesse:

http://127.0.0.1:8000/

Se tudo estiver correto, sua aplicação estará funcionando.

---

Resumo da ordem correta

1. Criar o ambiente virtual.
2. Ativar o ambiente virtual.
3. Instalar o Django.
4. Criar o projeto.
5. Configurar o "settings.py" (opcional).
6. Criar o app.
7. Registrar o app em "INSTALLED_APPS".
8. Criar as Views.
9. Configurar as URLs.
10. Criar a pasta "templates".
11. Criar os arquivos HTML.
12. Executar "makemigrations".
13. Executar "migrate".
14. Executar "runserver".
15. Acessar a aplicação pelo navegador.
